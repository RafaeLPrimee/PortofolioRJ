Déploiement WordPress avec Docker

Déploiement d'une instance WordPress complète containerisée avec Docker, utilisant MySQL comme base de données.

📦 Prérequis

Avant de commencer, assure-toi d'avoir :

  • Docker installé
  • Docker Compose installé (souvent inclus avec Docker Desktop)
  • Accès à un terminal

Vérification :

docker --version
docker compose version

📁 Création de l'environnement

Créer un dossier de projet :

mkdir wordpress-docker
cd wordpress-docker

⚙️ Création du fichier docker-compose.yml

Créer un fichier :

nano docker-compose.yml

Puis ajouter :

version: '3.8'

services:
  db:
    image: mysql:5.7
    container_name: wordpress_db
    restart: always
    environment:
      MYSQL_DATABASE: wordpress
      MYSQL_USER: wp_user
      MYSQL_PASSWORD: wp_password
      MYSQL_ROOT_PASSWORD: root_password
    volumes:
      - db_data:/var/lib/mysql

  wordpress:
    image: wordpress:latest
    container_name: wordpress_app
    depends_on:
      - db
    ports:
      - "8080:80"
    restart: always
    environment:
      WORDPRESS_DB_HOST: db:3306
      WORDPRESS_DB_USER: wp_user
      WORDPRESS_DB_PASSWORD: wp_password
      WORDPRESS_DB_NAME: wordpress
    volumes:
      - wordpress_data:/var/www/html

volumes:
  db_data:
  wordpress_data:

🚀 Lancement des containers

Dans le dossier :

docker compose up -d

Cela va :

  • Télécharger les images nécessaires
  • Créer les containers
  • Lancer WordPress + MySQL

🌐 Accès à WordPress

Ouvre ton navigateur :

http://localhost:8080

Tu verras l'écran d'installation de WordPress :

  • Choisir la langue
  • Configurer le site
  • Créer un utilisateur admin

🔍 Vérification

Lister les containers actifs :

docker ps

Tu dois voir :

  • wordpress_app
  • wordpress_db

🛠️ Commandes utiles

Arrêter les containers :

docker compose down

Voir les logs :

docker compose logs -f

Redémarrer :

docker compose restart

💾 Persistance des données

Les volumes Docker :

  • db_data → base de données MySQL
  • wordpress_data → fichiers WordPress

👉 Les données restent même si les containers sont supprimés.

🔐 Bonnes pratiques (recommandé)

  • Changer les mots de passe par défaut
  • Ne pas exposer MySQL publiquement
  • Ajouter un reverse proxy (Nginx)
  • Activer HTTPS (ex : Let's Encrypt)
  • Sauvegarder les volumes régulièrement

✅ Résultat

Tu disposes maintenant :

  • d'un WordPress fonctionnel
  • d'une base MySQL séparée
  • d'une architecture conteneurisée, portable et reproductible
